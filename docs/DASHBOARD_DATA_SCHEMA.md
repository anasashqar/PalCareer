# دليل إدارة بيانات لوحة التحكم (Admin Dashboard Hub) - PalCareer

هذا الدليل مبسط ومنظم لمساعدة المبرمج في بناء لوحة تحكم متوافقة تماماً مع قاعدة بيانات التطبيق.

---

## 1. مجموعة الوظائف (`jobs`)
هذا هو القسم الأهم لإضافة وتعديل الوظائف.

### الحقول الأساسية (إدخال نصي)
| الحقل في Firestore | النوع | الوصف | مثال |
| :--- | :--- | :--- | :--- |
| `company` | String | اسم الشركة | "شركة التقنية المتقدمة" |
| `applyUrl` | String | رابط التقديم الخارجي | "https://example.com/apply" |
| `iconId` | String | معرف الأيقونة | "it" أو "medicine" |

### الحقول ثنائية اللغة (Map)
*يجب توفير خانتين في لوحة التحكم (واحدة للعربي وواحدة للإنجليزي):*
| الحقل في Firestore | النوع | محتويات الخريطة |
| :--- | :--- | :--- |
| `title` | Map | `{"ar": "عنوان بالعربي", "en": "English Title"}` |
| `location` | Map | `{"ar": "رام الله", "en": "Ramallah"}` |
| `description` | Map | (اختياري) وصف الوظيفة بالتفصيل |
| `requirements` | Map | مصفوفة متطلبات: `{"ar": ["..", ".."], "en": ["..", ".."]}` |
| `responsibilities`| Map | مصفوفة مسؤوليات: `{"ar": ["..", ".."], "en": ["..", ".."]}` |

### خيارات القوائم المنسدلة (Dropdowns)
*يجب على المبرمج الالتزام بهذه القيم (Key) لتظهر في التطبيق:*

**1. نوع العمل (`jobType`):**
- `full_time` (دوام كامل)
- `part_time` (دوام جزئي)
- `remote` (عن بعد)
- `freelance` (عمل حر)
- `contract` (عقد)
- `internship` (تدريب)

**2. مستوى الخبرة (`level`):**
- `entry` (مبتدئ / خريج جديد)
- `mid` (خبرة متوسطة)
- `senior` (خبير)
- `lead` (قائد فريق / إدارة)

## 2. جدول الأقسام والتصنيفات (Static Taxonomy)
هذه القيم هي التي تُستخدم في حقول `categoryId` و `subCategoryId`.

| القسم الرئيسي (`categoryId`) | الأقسام الفرعية التابعة له (`subCategoryId`) |
| :--- | :--- |
| **تقنية المعلومات** (`it`) | تطوير برمجيات (`software_dev`), تطبيقات موبايل (`mobile_dev`), تطوير ويب (`web_dev`), بيانات وذكاء اصطناعي (`data_ai`), شبكات وحماية (`networks`), تصميم واجهات (`design_ux`) |
| **الهندسة** (`engineering`) | هندسة مدنية (`civil`), هندسة معمارية (`architecture`), هندسة كهربائية (`electrical`), هندسة ميكانيكية (`mechanical`) |
| **الطب والصحة** (`medicine`) | طب عام (`general_medicine`), تمريض (`nursing`), صيدلة (`pharmacy`), طب أسنان (`dentistry`), علاج طبيعي (`physical_therapy`) |
| **الأعمال والمالية** (`business`) | محاسبة (`accounting`), تسويق ومبيعات (`marketing`), موارد بشرية (`hr`), إدارة مشاريع (`management`) |
| **التعليم** (`education`) | تدريس (`teaching`), إرشاد تربوي (`counseling`), تربية خاصة (`special_ed`) |

---

## 3. مجموعة المستخدمين (`users`)
لعرض بيانات المستخدمين المسجلين وتفضيلاتهم.

| الحقل | الوصف |
| :--- | :--- |
| `displayName` | اسم المستخدم كامل |
| `email` | البريد الإلكتروني |
| `preferredCategoryIds`| مصفوفة الأقسام المفضلة (مثل: `["it", "business"]`) |
| `preferredJobTypes` | مصفوفة أنواع الدوام المفضلة (مثل: `["full_time", "remote"]`) |
| `isActive` | (Boolean) هل الحساب نشط |

---

## ملاحظات فنية للمبرمج:
1. **التواريخ:** استخدم `Timestamp` لـ `postedAt` (تاريخ النشر) و `expiresAt` (تاريخ الانتهاء).
2. **الحالة:** `isActive` في الوظائف يجب أن تكون `true` لتظهر للمستخدمين.
3. **التصنيفات:** إذا أردت إضافة قسم جديد، يجب تحديث مستند `app_config/career_taxonomies` في Firestore أولاً.
