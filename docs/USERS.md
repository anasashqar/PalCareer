## 🎓 Seeker — الطالب / الخريج الجديد

### من هو؟

- **الفئة العمرية**: 19–26 سنة
- **الموقع الجغرافي**: فلسطين (الضفة الغربية وغزة)
- **الحالة**: طالب جامعي في سنواته الأخيرة، أو حديث التخرج (0–2 سنة خبرة)
- **اللغة**: يتعامل بالعربية بشكل رئيسي، مع مستوى مقبول من الإنجليزية
- **الجهاز**: هاتف Android (غالباً مستوى متوسط)

### ماذا يريد؟
- أن يجد وظيفة تناسب **تخصصه الجامعي** وليس فقط مستواه
- أن يكون **أول من يعلم** عند نشر وظيفة تناسبه
- تجربة بسيطة وسريعة — **لا وقت لبحث طويل**
- عدم رؤية وظائف **لا علاقة لها باهتماماته**

### نقاط الألم (Pain Points)
- المواقع الحالية مزدحمة ولا تُصفي الوظائف بشكل ذكي
- الوظائف تُنشر ويتأخر في معرفتها
- كثير من الوظائف لا تُحدد الخبرة المطلوبة بوضوح
- الواجهات معقدة أو غير محسّنة لهاتف Android متوسط

### 🗺️ رحلة المستخدم — Seeker

#### 1️⃣ الوصول الأول
```
يرى التطبيق (من صديق / APK / متجر)
        ↓
يفتح التطبيق لأول مرة
        ↓
[Splash Screen] — يرى الشعار لثانيتين
        ↓
[Login Screen] — يرى الشعار + عداد الوظائف + زر Google
        ↓
يضغط "تسجيل الدخول بـ Google" ويختار حسابه
        ↓
[Onboarding Step 1] — يختار تخصصه (أو أكثر)
        ↓
[Onboarding Step 2] — يختار مستواه التعليمي + نوع الدوام المفضل
        ↓
يضغط "ابدأ" → تُحفظ البيانات في Firestore
        ↓
[الشاشة الرئيسية] — يرى وظائف مخصصة له فوراً
```

#### 2️⃣ الاستخدام اليومي
```
يصله إشعار: "وظيفة جديدة: Flutter Developer في رام الله"
        ↓
يضغط على الإشعار
        ↓
[تفاصيل الوظيفة] — يقرأ المتطلبات والمسؤوليات
        ↓
يضغط "تقدم الآن" → يُفتح رابط التقديم في المتصفح
```

#### 3️⃣ البحث الذاتي
```
يفتح التطبيق
        ↓
[الشاشة الرئيسية] — يرى وظائف مخصصة مرتبة من الأحدث
        ↓
يستخدم شريط الفلاتر للتصفية (تخصص محدد / دوام عن بُعد)
        ↓
يضغط على وظيفة تستهويه
        ↓
[تفاصيل الوظيفة] → يقرأ ثم يتقدم
```

#### 4️⃣ تعديل التفضيلات
```
يلاحظ أن الوظائف غير مناسبة (غيّر اهتمامه)
        ↓
يذهب لـ [الملف الشخصي]
        ↓
يضغط "تعديل الملف الشخصي"
        ↓
يغيّر تخصصاته أو نوع الدوام
        ↓
يحفظ → تُحدَّث قائمة الوظائف تلقائياً
```

## 🛠️ Admin — مدير النظام

### من هو؟
- المطوّر نفسه (في المرحلة الأولى)
- لديه صلاحية الوصول الكاملة إلى Firebase Console
- يجمع الوظائف يدوياً من مواقع مثل LinkedIn, Bayt, وغيرها
- يُدخل الوظائف بشكل منظم في Firestore

### ماذا يريد؟
- إدخال وظيفة جديدة بسرعة ودون تعقيد
- التأكد من وصول الإشعار للمستخدمين المناسبين
- التحكم في ظهور الوظيفة أو إخفائها (`isActive`)

### رحلة الـ Admin — V1
```
يفتح Firebase Console
        ↓
يذهب إلى Firestore → Collection "jobs"
        ↓
يُنشئ Document جديد بالحقول المطلوبة (بلغتين)
        ↓
يضبط isActive: true
        ↓
Cloud Function تُشغَّل تلقائياً
        ↓
يصل FCM لكل مستخدم تطابق اهتماماته
```

## 📊 ملف المستخدم في Firestore (Seeker)

```
users/{userId}
├── name: String                         // من Google Account
├── email: String                        // من Google Account
├── photoUrl: String                     // من Google Account
├── fields: Array<String>               // ["it", "engineering"]
├── educationLevel: String              // "student" | "graduate"
├── preferredJobTypes: Array<String>    // ["full_time", "remote"]
├── fcmToken: String                    // يُحدَّث عند كل دخول
├── onboardingCompleted: Boolean
└── createdAt: Timestamp
```

## 💬 ملاحظات للمصمم والمطوّر

- غالبية المستخدمين يستخدمون **هواتف متوسطة** — تجنب المؤثرات الثقيلة
- الجمهور يتحدث العربية أولاً — الواجهة الافتراضية يجب أن تكون **RTL عربية خالصة**
- وقت الانتباه قصير — الشاشة الرئيسية يجب أن تعرض **قيمة فورية** (وظائف مخصصة في أول فتحة)
- الثقة مهمة — الاسم الفلسطيني والهوية المحلية تُعزّز التبني من المستخدمين
